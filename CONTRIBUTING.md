# 貢獻指南

感謝你有興趣參與台灣交通改革協作平台！以下說明如何貢獻。

## 📝 回報道路問題（開 Issue）

### 必填資訊

| 欄位 | 說明 |
|------|------|
| 地點 | 具體地址、路口名稱或 Google Maps 連結 |
| 縣市 | 從下拉選單選擇 |
| 問題類型 | 從下拉選單選擇，若無適合選項請選「其他」 |
| 問題描述 | 詳細說明觀察到的問題 |

### 建議提供

- 📸 現場照片（強烈建議）
- 🏛️ 負責單位（如知道）
- 📋 其他資訊（附近學校/醫院、曾發生事故等）

### 好的 Issue 範例

> **標題**：復興南路一段人行道破損
> 
> **地點**：台北市大安區復興南路一段 23 號前
> 
> **問題描述**：人行道磁磚大面積破損隆起，輪椅無法通行，多位長者曾跌倒。
> 
> **照片**：[附上清楚照片]

## ✅ 回報已解決的問題（提 PR）

### 前提條件

- 原 Issue 存在且狀態為「已陳情」或「處理中」
- 問題確實已被改善
- 你有改善後的照片

### 步驟

1. **Fork** 這個 repo
2. 在 `resolved/` 資料夾建立檔案

   **命名格式**：`YYYY-MM-縣市-簡短描述.md`
   
   **範例**：`2026-01-台北市-復興南路人行道.md`

3. 使用以下模板填寫內容：

```markdown
# [地點] [問題簡述]

| 項目 | 內容 |
|------|------|
| **原始 Issue** | #123 |
| **地點** | 完整地址 |
| **問題類型** | 人行道/行穿線/... |
| **通報日期** | YYYY-MM-DD |
| **解決日期** | YYYY-MM-DD |
| **負責單位** | XX市政府XX局 |

## 問題描述

（簡述原有問題）

## 改善內容

（說明如何改善）

## 照片對比

| 改善前 | 改善後 |
|--------|--------|
| ![before](連結) | ![after](連結) |
```

4. 提交 PR，**PR 描述中使用 `Closes #123` 關聯原 Issue**

5. 等待 **5 位參與者** 確認後合併

### PR 審核標準

審核者請確認：

- [ ] 照片清楚顯示改善成果
- [ ] 地點與原 Issue 一致
- [ ] 改善內容符合原問題描述

確認無誤請回覆 `✅ 確認已解決` 或使用 GitHub Approve。

## 🏷️ Label 說明

### 狀態標籤

| 標籤 | 說明 | 誰可以加 |
|------|------|---------|
| `狀態/待確認` | 新開的 Issue | 自動 |
| `狀態/已確認` | 問題已確認存在 | Maintainer |
| `狀態/已陳情` | 已向政府反映 | Maintainer |
| `狀態/處理中` | 政府處理中 | Maintainer |
| `狀態/已解決` | PR 合併後自動加上 | 自動 |
| `狀態/無法解決` | 被駁回或無解 | Maintainer |

### 縣市標籤

根據 Issue 模板自動加上，例如 `縣市/台北市`。

### 類型標籤

| 標籤 | 說明 |
|------|------|
| `類型/人行道` | 人行道缺失或破損 |
| `類型/行穿線` | 行穿線設計不良 |
| `類型/標線標誌` | 標線或標誌問題 |
| `類型/號誌` | 紅綠燈號誌問題 |
| `類型/自行車道` | 自行車道問題 |
| `類型/無障礙` | 無障礙設施不足 |
| `類型/其他` | 不在上述分類中 |

## ❓ 常見問題

### Q: 我發現問題已解決但沒人提 PR？

你可以自己提！只要附上改善後的照片即可。

### Q: Issue 的負責單位我不知道怎麼辦？

沒關係，留空即可，社群或 Maintainer 會協助補充。

### Q: PR 需要多久才會被合併？

達到 5 人確認後會盡快合併。如果超過一週沒有動靜，歡迎在 PR 中 @maintainer。
